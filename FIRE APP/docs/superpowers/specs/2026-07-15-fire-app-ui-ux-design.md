## 1. 设计原则

### 1.1 设计理念

FIRE 计算APP 是一款面向个人用户的桌面端财务管理与 FIRE 退休规划工具。UI/UX 设计遵循三大核心理念：

| 理念 | 说明 | 设计体现 |
|------|------|---------|
| **专业感** | 传递金融工具的可信度和严谨性 | 深色侧边栏 + 浅色内容区，数据表格密集排列，数字使用等宽字体 |
| **数据驱动** | 以数据可视化和数值展示为核心 | 每个页面都有概览卡片或图表，关键数值突出显示，趋势一目了然 |
| **简洁克制** | 去除装饰性元素，聚焦信息传达 | 无渐变背景、无多余图标动画、卡片阴影极浅，留白充足 |

**设计哲学**：工具应消失在用户的工作流中。用户打开 APP 是为了查看财务状况或记录交易，UI 不应成为阻碍。每个页面遵循"概览 → 明细 → 操作"三段式信息架构，用户无需思考即可找到所需功能。

**目标用户画像**：关注 FIRE（财务独立、提前退休）的中国个人投资者，熟悉基本的财务概念（资产分类、投资回报率、提现率），使用 Windows 10 桌面环境，期望高效管理个人财务并规划退休路径。

### 1.3 字体规范

#### 1.3.1 字体族

```css
/* 全局字体栈 / Global font stack */
font-family: 'PingFang SC', 'Microsoft YaHei', 'Inter', system-ui, -apple-system, sans-serif;

/* 数字等宽字体栈 / Monospace number font stack */
font-variant-numeric: tabular-nums;
```

- **中文字体**：`PingFang SC`（macOS）、`Microsoft YaHei`（Windows 10 默认），覆盖目标用户的桌面环境。
- **英文/数字字体**：`Inter`，设计清晰、数字等宽特性好，适合金融数据展示。
- **数字等宽**：所有金额、百分比等数值使用 `tabular-nums`，保证表格中数字列对齐。

### 1.5 响应式断点

FIRE 计算APP 为 Electron 桌面应用，窗口大小可由用户自由拖拽调整。设计以固定最小宽度为基准，不涉及移动端断点（移动端适配为后续扩展）。

| 断点名称 | 窗口宽度 | 说明 |
|---------|---------|------|
| minimum | 1024px | 应用最小窗口宽度（`minWidth: 1024` 在 BrowserWindow 中设置），低于此宽度出现横向滚动条 |
| compact | 1024px – 1280px | 紧凑布局：侧边栏折叠为图标模式（64px 宽），内容区单列 |
| recommended | 1280px – 1440px | 推荐布局：侧边栏展开（200px 宽），内容区双列（概览 + 明细） |
| spacious | 1440px+ | 宽松布局：内容区可三列，图表区域更大 |

**默认窗口尺寸**：`width: 1280, height: 800`（在 BrowserWindow 配置中设定），启动即为推荐布局。

**侧边栏自适应**：
- 窗口宽度 ≥ 1280px：侧边栏展开，宽度 200px，显示图标 + 文字标签。
- 窗口宽度 < 1280px：侧边栏折叠，宽度 64px，仅显示图标，hover 时显示文字 tooltip。

## 2. 导航架构

### 2.1 侧边栏导航结构

侧边栏位于窗口左侧，采用深色背景（`blue-900`），包含 5 个主导航项和 1 个底部用户信息区。

|------|--------|---------|---------|------|
| 1 | 账户管理 | `/accounts` | 钱包图标：圆角矩形内含币种符号 | 管理所有金融账户，查看资产概览 |
| 2 | 交易记录 | `/transactions` | 双向箭头图标：左右箭头交叉 | 记录和查看所有收支、转账交易 |

**侧边栏布局**（自上而下）：
- 顶部：APP Logo 区（高度 56px），显示"FIRE"文字 Logo + 火焰图标

- 底部：用户信息区（高度 56px），显示用户头像（首字母圆形）+ `display_name`

### 2.3 导航状态管理

**当前选中页高亮方案**：

```typescript
// 导航项高亮状态通过 React Router 的 useLocation hook 判断
// Navigation highlight state determined by React Router's useLocation hook

import { useLocation } from 'react-router-dom';

function NavItem({ to, icon, label }: NavItemProps) {
  const location = useLocation();
  const isActive = location.pathname === to ||
    (to !== '/accounts' && location.pathname.startsWith(to));

  // 选中态样式：
  // - 背景：blue-600 (#2563EB)
  // - 文字：white (#FFFFFF)
  // - 左侧 3px 蓝色高亮条
  // 未选中态样式：
  // - 背景：透明
  // - 文字：blue-300 (#93C5FD)
  // - hover 背景：blue-800/50 (rgba 半透明)

  return (
    <NavLink
      to={to}
      className={`flex items-center gap-3 px-4 h-11 rounded-md transition-colors ${
        isActive
          ? 'bg-blue-600 text-white border-l-[3px] border-white'
          : 'text-blue-300 hover:bg-blue-800/50'
      }`}
    >
      <span className="w-5 h-5">{icon}</span>
      <span className="text-body">{label}</span>
    </NavLink>
  );
}
```

**导航状态存储**：导航高亮完全依赖 URL 路径（React Router `useLocation`），不额外存储在 Zustand Store 中。这保证了刷新页面后导航状态正确恢复，且支持浏览器前进/后退。

| 卡片 | 数据来源 | 计算方式 |

| 投资资产合计 | `Account.current_balance` WHERE `asset_class='invested'` | `SUM()` |
| 使用资产合计 | `Account.current_balance` WHERE `asset_class='use_asset'` | `SUM()` |
| 负债合计 | `Account.current_balance` WHERE `asset_class='liability'` | `SUM()`（负数） |
| 净资产 | 所有账户 | `SUM(current_balance)` |

#### 3.1.4 关键交互说明

1. **新增账户**：点击右上角"新增账户"按钮 → 弹出账户表单模态框 → 填写表单 → 提交 → 关闭模态框 + 刷新列表和概览卡片。
2. **编辑账户**：点击表格行"编辑"按钮 → 弹出预填充的账户表单模态框 → 修改 → 提交 → 刷新列表。
3. **删除账户**：点击表格行"删除"按钮 → 弹出确认对话框 → 确认删除 → 若有关联交易，显示错误 Toast"该账户下有 N 笔交易，无法删除" → 无关联交易则软删除 + 刷新列表。
4. **表格排序**：点击表头可按"账户名称"、"当前余额"排序，默认按 `display_order` 排序。

### 5.5 图表库选型结论

**确认选型：Recharts v2.x**

选型理由（与前端架构文档 ADR-015 一致）：

1. **React 原生组件式**：Recharts 以 React 组件形式声明图表（`<LineChart>`、`<AreaChart>` 等），与 React 19 生命周期完美集成，无需手动管理 DOM。
2. **响应式支持**：`<ResponsiveContainer>` 组件自动适配父容器尺寸，适合 Electron 窗口大小可变的场景。
3. **覆盖全部需求**：折线图（净资产趋势）、环形图（资产配比）、面积图（FIRE 投影）、径向条图（进度仪表盘）均原生支持。
4. **TypeScript 支持**：Recharts v2 提供完整的 TypeScript 类型定义，与项目 TypeScript-first 策略一致。
5. **轻量级**：打包体积约 100KB（gzip 后约 35KB），对 Electron 桌面应用性能影响极小。

**Recharts 组件使用方案**：

| 图表场景 | Recharts 组件组合 |
|---------|------------------|
| 净资产趋势图 | `<ResponsiveContainer>` + `<LineChart>` + `<Line>` + `<XAxis>` + `<YAxis>` + `<Tooltip>` + `<CartesianGrid>` |
| 资产配比图 | `<ResponsiveContainer>` + `<PieChart>` + `<Pie>` + `<Cell>` + `<Legend>` + `<Tooltip>` |
| FIRE 投影图 | `<ResponsiveContainer>` + `<AreaChart>` + `<Area>` × 2 + `<XAxis>` + `<YAxis>` + `<Tooltip>` + `<ReferenceLine>` + `<CartesianGrid>` |
| 进度仪表盘 | `<ResponsiveContainer>` + `<RadialBarChart>` + `<RadialBar>` + 自定义 `<text>` 中心标签 |

**封装组件**：所有图表封装为 `<ChartContainer>` 组件的子组件，统一管理加载状态、空状态和错误处理。

- 错误文字前显示警告图标（12px）

#### 7.3.3 错误码到中文消息映射表

| 错误码 (IpcError.code) | 触发场景 | 中文消息 | Toast 类型 |
|----------------------|---------|---------|-----------|
| `VALIDATION_ERROR` | 表单校验失败 | 输入有误，请检查后重试 | error |
| `VALIDATION_ERROR` | 金额 ≤ 0 | 金额必须大于 0 | error |
| `VALIDATION_ERROR` | 退休年龄 ≤ 当前年龄 | 退休年龄必须大于当前年龄 | error |
| `VALIDATION_ERROR` | 提现率超出范围 | 提现率应在 2%-6% 范围内 | error |
| `VALIDATION_ERROR` | 转账目标账户为空 | 转账交易必须选择转入账户 | error |
| `VALIDATION_ERROR` | 转账给自己 | 不能转账给同一账户 | error |
| `NOT_FOUND` | 账户不存在 | 账户不存在或已被删除 | error |
| `NOT_FOUND` | 交易不存在 | 交易不存在或已被删除 | error |
| `NOT_FOUND` | 分类不存在 | 分类不存在或已被删除 | error |
| `NOT_FOUND` | 场景不存在 | 场景不存在或已被删除 | error |
| `DB_ERROR` | 删除有关联交易的账户 | 该账户下有 {N} 笔交易，无法删除。请先处理关联交易 | error |
| `DB_ERROR` | 删除有关联交易的分类 | 该分类下有 {N} 笔交易，无法删除。请先迁移关联交易 | error |
| `DB_ERROR` | 账户名称重复 | 已存在同名账户，请使用其他名称 | error |
| `DB_ERROR` | 事务失败 | 操作失败，数据未更改。请重试 | error |
| `DB_ERROR` | 数据库连接失败 | 数据库连接失败，请重启应用 | error |

**错误处理流程**：
1. 渲染进程调用 `dataAccess` 方法 → Promise reject
2. Store 的 action 中 catch 错误，提取 `IpcError.code` 和 `message`
3. 查询错误码映射表，获取中文消息
4. 调用 `toastStore.showToast('error', chineseMessage)` 显示 Toast
5. 设置 Store 的 `error` 状态字段（部分场景需要）
